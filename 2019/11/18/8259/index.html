中断方式

中断概述

  1. 中断：CPU暂停现行程序，转而处理随机到来的事件。

  2. 基本模型

  3. 中断过程：

    1. 中断响应：
      1. 在每条指令的最后一个时钟周期，CPU检测INTR或NMI信号。
      2. 断点保护：
        1. 将OSW的内容压入堆栈。
        2. 自动关中断：IF=0,TF=0
        3. 将CS的内容压入堆栈。
        4. 将IP的内容压入堆栈。
      3. 中断判优：
        1. 优先处理优先级较高的中断。如果当前正在执行低优先级任务，那么允许它会被高优先级的中断源所中断—中断嵌套。
        2. 优先级相同，则遵照先来先服务原则。
      4. 识别中断源
        1. 软件查询：将中断信号从数据总线读入，用程序进行判断。
        2. 中断矢量法：由中断源提供中断类型，CPU根据类型确定中断源。
      5. 获得中断服务程序的首地址
        1. 中断服务子程序的工作：
          1. 保护现场(PUSH reg)
          2. 开中断(STI)
          3. 进行中断处理
          4. 关中断(CLI)
          5. 恢复现场(POP reg)
          6. 中断返回(IRET)
      6. 中断返回：执行中断返回指令IRET，将内存中保存的断点信息弹出到IP、CS和FLAG中。

Intel 16位中断系统

x86中断响应过程

  1. Flags(标志寄存器)、CS(代码段寄存器)、IP(指令指针寄存器)压入栈，关中断(CLI)
  2. 第一次INTA，通知CPU要进行中断。
  3. 第二次INTA，取中断向量码。
  4. 取得中断向量地址：由于一个中断向量占4B，所以要乘4。
  5. 在中断向量中得到基址放入IP，得到偏移放入(CS)
  6. 跳转到CS和IP共同指向的地址执行中断处理程序。
  7. 执行到IRET（中断返回）时返回。弹出栈中IP,CS,Flags。

8259

  1. 内部结构
    1. 中断请求寄存器IRR：某位为1代表对应位有中断。
    2. 中断服务寄存器ISR:某位为1代表对应位正在被服务。
    3. 中断屏蔽寄存器IMR：某位为1代表不应该为该信号向CPU发中断。
  2. 引脚
  3. 8259工作方式：
    1. 中断结束方式：
      1. 非自动：通过中断处理程序提供EOI（特殊/一般）
      2. 自动：在第2个INTA后沿自动执行。
    2. 缓冲方式：缓冲/非缓冲
    3. 嵌套方式
      1. 一般嵌套：单片/从片
      2. 特殊全嵌套：仅主片
    4. 屏蔽方式：一般屏蔽，特殊屏蔽
    5. 优先级规定
      1. 固定优先级：0最大
      2. 循环优先级 自动/指定
  4. 嵌套方式：
    1. 一般嵌套方式：只有更高优先级的事件可以打断当前中断处理过程而被服务
    2. 特殊全前嵌套方式：一中断正被处理时，允许同级或更高优先级的事件打断当前的中断处理过程而被服务。
  5. 中断方式
    1. 自动EOI：因不保留当前正在服务的中断状态不能用于中断嵌套方式。
    2. SEOI：由CPU发出一条SEOI命令，该EOI命令指出了所要复位的ISR信号。用于特殊屏蔽方式
    3. NSEOI ：由CPU发出正常EOI命令，使得ISRi=1中优先级最高的那一位复位。用于一般屏蔽方式
    4. 在特殊全嵌套下，只有当从属PIC的中断全部处理完后，才向主PIC发EOI命令。
  6. 优先级规定
    1. 固定优先级：0最大，7最小。
    2. 循环优先级：
      1. 自动循环优先级：初始优先级可编程，之后当某个中断请求被处理后，其优先级自动将为最低，原本它的下一优先级中断升为最高级。
    3. 指定优先级：通过OCW2

8259级联

  1. 最多支持64个中断源，n片级联支持7n+1个中断源。

  2. 级联方式见本

  3. 中断响应方式

    1. 主控8259：在第一个INTA周期，清IRR相应位， 设置ISR相应位，送级联地址CAS0～CAS2。
    2. 从属8259：读级联地址，若与识别地址（ICW3） 一致，则接收INTA信号，清除相应IRR位、置位 相应ISR位；在第二个INTA周期将中断向量码 （ICW2）输出至数据总线。
    3. 若中断源直接连在主控8259的IR端（由ICW3确 定），则中断向量由主控8259提供。

  4. 8259编程

    1. 初始化过程，向ICW1-ICW4写入。

    2. 操作方式编程：向OW1-OW3写入。

初始化编程

  1. ICW1 初始化字
    ICW1写入后：清除ISR,IMR；将中断优先级设成初始默认；设定为一般屏蔽方式；采用非自动中断结束方式；状态读出逻辑阈值为IRR

    1. LTIM(触发方式)：=1高电平触发；=0上升沿触发；
    2. SNGL（级联控制）=1单片；=0级联；
    3. IC4（ICW4控制）：=1写ICW4;=0不写ICW4；

  2. ICW2 中断向量码

    1. D7-D3：为中断向量码的高5位，D2-D0为中断源序号

  3. ICW3 级联控制字

    1. 主片的ICW3中Si=1代表对应线上连接了从片 2. 从片D2-D0说明连接到主片的哪个引脚上。 3. 中断响应时，主片通过级联线CAS2-CAS0送出被允许中断的从片标识码，各从片用自己的ICW3与CAS2-CAS0比较，二者一样从片才发送中断向量码。

  4. ICW4 中断结束方式字

    1. SFNM =1特殊全嵌套；=0一般嵌套； 2. AEOI =1自动EOI方式；=0非自动EOI方式； 3.

操作方式编程

OCW1必须写入奇地址（A0=1）

OCW2，OCW3必须写入偶地址(A0=0)

  1. OCW1 中断屏蔽字

    1. 其中Mi=1代表IRi被屏蔽；=0允许IRi中断。
    2. A0=1时，读OCW1可读出设置。

  2. OCW2 中断结束和优先级循环

    1. L2-L0 指定优先级/EOI编码
    2. R 循环优先级
    3. SL 只当优先级/只当EOI
    4. EOI 执行结束中断命令

  3. OCW3 屏蔽方式和读出控制字

    1. 读IRR/ISR 写入此命令后，对同一地址，即可得到IRR和ISR内容。
    2. 比如，CPU将D2=1写入OCW3，再对同一地址读入，如果I=1代表有中断，中断请求号为R2-R0；可反复查询。
