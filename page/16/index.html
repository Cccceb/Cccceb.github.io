UVA 12258 埃及分数

输入出a,b,k，然后输入k个数，要求这k个数不出现在结果中。埃及分数是将任何一个分数a/b，转换成1/x1+1/x2+1/x3…的形式。好的解应该是项数最少的，对于相同项数的解，最小的分数值越大越好。

思路

  1. 由于本题中深度没有上限，而且加数的选择也没有上届，因此如果用广度有限遍历，可能连一层都遍历不完。
  2. 使用迭代加深度搜索解决。不断枚举深度的上限maxd，每次搜索深度不超过maxd的节点。这样只要解的深度有限，那么一定可以在有限的时间内枚举得到；
  3. 关于剪枝：本题中，如果剩余的maxd-d+1项都是1/i，但是和仍然小于目标a/b，那么说明项数不够，则不需要继续递归下去。
  4. 关于约分：使用到gcd算法，求出g = gcd(a,b)，那么a/b 可以转换成(a/g)/(b/g)；
  5. 关于inline:使用inline之后，时间从2290减到1380。不能使用inline的情况包括
    1. 不适用于包含复杂控制语句的函数
    2. 不适用于直接递归函数

但是是否真的设置为内联，由编译器决定

UVA 12325 宝箱

你有一个体积为N的箱子和两种数量无限的宝物。宝物1的体积为S1，价值为V1:；宝物2的体积为S2，价值为V2。输入均为32位带符号整数。 计算最多能装多大价值的宝物。（每种宝物都必须拿非负整数个）。

思路

  1. 可以选择枚举宝藏1的个数，然后尽可能多的拿宝物2；或者；但是当n和s1,s2的差距较大的时候，这种方法不奏效。
  2. 假设宝物1取总体积S2，宝物2取总体积S1，且二者总价值相等，即S1V2 == S2V1。那么当V1较大时，宝藏1至多取S2-1个，否则可用S1个宝物二替换。同理，如果V2较大，那么宝物2至多取S1-1个.因此，枚举量只有S1和S2。
  3. 解决TLE：
    1. 当N/S1比较小时，枚举宝物1的个数，时间复杂度为O(N/S1)；否则，当N/S2比较小时，枚举宝物2的个数，时间复杂度为O(N/S2)。
    2. 因此枚举的时间复杂度为O(max(S1,S2))

UVA 11212 编辑书稿

你有一篇n(2≤n≤9)个自然段组成的文章，希望将它们排列成1，2，…，n。可以用Ctrl+X（剪切）和Ctrl+V（粘贴）快捷键来完成任务。每次可以剪切一段连续的自然段，粘贴时按照顺序粘贴。注意，剪贴板只有一个，所以不能连续剪切两次，只能剪切和粘贴交替。例如，为了将{2，4，1，5，3，6}变为升序，可以剪切1将其放到2前，然后剪切3将其放到4前。再如，排列{3，4，5，1，2}，只需一次剪切和一次粘贴即可——将{3，4，5}放在{1，2}后，或者将{1，2}放在{3，4，5}前。

思路

  1. 典型的状态空间搜索问题，其中的状态是1-n的排列。排列最多有9！个。但是由于剪切和粘贴的位置不定，所以搜索空间很大。

  2. 关于剪枝

    1. 每次只剪切一段连续的数字，不破坏已经有序的串。
    2. 每次剪切至多使后继数字不正确的个数减少3个。考虑abc变成acb时，a b c 三段的末尾的数字。

  3. 迭代加深搜。迭代以获得下一次递归调用的初始状态，深度搜索来找解。迭代的时候需要枚举剪切的长度，剪切和粘贴的起始位置。注意循环的边界问题。

  4. 进行剪切和粘贴的时候，要分情况，因为涉及到其他元素往前移动还是往后移动的问题。

    1. 如果start位置在pos后面，那么粘贴之后，需要将pos之后到start的元素向后移动len位。
    2. 同理如果start在pos前面，则需要从pos开始到start之前的元素向前移动len位。

UVA 816 ABBOTT的复仇

给出一个箭头迷宫，在每个路口处，如果你从某个方向进入了该路口，那么路口的地面上在靠近你的方向会画有一组箭头，它们相对于你的方向可以是向左，向前，向右，或者是它们的任意组合。

当你从某一方向进入某个入口时，下一步只能在这个入口对应方向上标记的箭头中选一个方向继续行进。在起点时，可以选择任何方向。

给出起点和终点，求它们之间的最短路径。

每条边的长度为1

思路

  1. 对于迷宫问题，一般采用遍历的方法去找出口，本题中采用广度优先遍历。
  2. 本题特殊在进入每个交叉点的方向不同，允许他出去的方向也不同。采用map来存储进入方向和出去方向的对应关系，因为一个进入方向对应的出去方向不唯一，所以采用map<int,set> 来存储
  3. 每个位置不同进入方向的转向也可以是用二进制法记录
  4. 本题主要是细节问题比较多。并且因为图的大小至多为9*9，所以也可以使用高维数组

中断方式

中断概述

  1. 中断：CPU暂停现行程序，转而处理随机到来的事件。

  2. 基本模型

  3. 中断过程：

    1. 中断响应：
      1. 在每条指令的最后一个时钟周期，CPU检测INTR或NMI信号。
      2. 断点保护：
        1. 将OSW的内容压入堆栈。
        2. 自动关中断：IF=0,TF=0
        3. 将CS的内容压入堆栈。
        4. 将IP的内容压入堆栈。
      3. 中断判优：
        1. 优先处理优先级较高的中断。如果当前正在执行低优先级任务，那么允许它会被高优先级的中断源所中断—中断嵌套。
        2. 优先级相同，则遵照先来先服务原则。
      4. 识别中断源
        1. 软件查询：将中断信号从数据总线读入，用程序进行判断。
        2. 中断矢量法：由中断源提供中断类型，CPU根据类型确定中断源。
      5. 获得中断服务程序的首地址
        1. 中断服务子程序的工作：
          1. 保护现场(PUSH reg)
          2. 开中断(STI)
          3. 进行中断处理
          4. 关中断(CLI)
          5. 恢复现场(POP reg)
          6. 中断返回(IRET)
      6. 中断返回：执行中断返回指令IRET，将内存中保存的断点信息弹出到IP、CS和FLAG中。

Intel 16位中断系统

x86中断响应过程

  1. Flags(标志寄存器)、CS(代码段寄存器)、IP(指令指针寄存器)压入栈，关中断(CLI)
  2. 第一次INTA，通知CPU要进行中断。
  3. 第二次INTA，取中断向量码。
  4. 取得中断向量地址：由于一个中断向量占4B，所以要乘4。
  5. 在中断向量中得到基址放入IP，得到偏移放入(CS)
  6. 跳转到CS和IP共同指向的地址执行中断处理程序。
  7. 执行到IRET（中断返回）时返回。弹出栈中IP,CS,Flags。

8259

  1. 内部结构
    1. 中断请求寄存器IRR：某位为1代表对应位有中断。
    2. 中断服务寄存器ISR:某位为1代表对应位正在被服务。
    3. 中断屏蔽寄存器IMR：某位为1代表不应该为该信号向CPU发中断。
  2. 引脚
  3. 8259工作方式：
    1. 中断结束方式：
      1. 非自动：通过中断处理程序提供EOI（特殊/一般）
      2. 自动：在第2个INTA后沿自动执行。
    2. 缓冲方式：缓冲/非缓冲
    3. 嵌套方式
      1. 一般嵌套：单片/从片
      2. 特殊全嵌套：仅主片
    4. 屏蔽方式：一般屏蔽，特殊屏蔽
    5. 优先级规定
      1. 固定优先级：0最大
      2. 循环优先级 自动/指定
  4. 嵌套方式：
    1. 一般嵌套方式：只有更高优先级的事件可以打断当前中断处理过程而被服务
    2. 特殊全前嵌套方式：一中断正被处理时，允许同级或更高优先级的事件打断当前的中断处理过程而被服务。
  5. 中断方式
    1. 自动EOI：因不保留当前正在服务的中断状态不能用于中断嵌套方式。
    2. SEOI：由CPU发出一条SEOI命令，该EOI命令指出了所要复位的ISR信号。用于特殊屏蔽方式
    3. NSEOI ：由CPU发出正常EOI命令，使得ISRi=1中优先级最高的那一位复位。用于一般屏蔽方式
    4. 在特殊全嵌套下，只有当从属PIC的中断全部处理完后，才向主PIC发EOI命令。
  6. 优先级规定
    1. 固定优先级：0最大，7最小。
    2. 循环优先级：
      1. 自动循环优先级：初始优先级可编程，之后当某个中断请求被处理后，其优先级自动将为最低，原本它的下一优先级中断升为最高级。
    3. 指定优先级：通过OCW2

  1. GATE：上升沿开始计数。
  2. OUTPUT:产生n*T的周期性正脉冲。
